# 错误码与异常规范

本文是错误码、HTTP 响应和异常映射的登记基线。仓库内存在已审批的 `docs/错误码及异常设计.md` 时，以该文件为项目登记源。

## 1. REST 响应

所有对客户端公开的 REST 接口返回 `R<T>`；文件下载、流式响应和 WebSocket 握手等例外必须在 OpenAPI 中声明。Feign 和 MQ 不使用 `R<T>`。

```json
{
  "code": 0,
  "msg": "success",
  "data": {},
  "traceId": "01J..."
}
```

- `0` 是唯一成功码；失败必须使用已登记的六位业务码，不得使用 `500`、`404` 等 HTTP 数字充当业务码。
- `msg` 使用错误码绑定的稳定、脱敏文案，不返回堆栈、SQL、Token 或个人信息。
- 失败时 `data` 默认是 `null`；参数校验失败可使用约定的字段错误数组。
- HTTP 状态表达传输语义，业务码表达业务语义；不得用 HTTP 200 包装所有失败。
- 网关产生的 401、403、429、503 必须返回同一 JSON 结构。WebFlux 与 MVC 异常处理不得混用 Servlet/Reactive API。
- `R<T>` 只提供 `fail(IErrorCode)`；禁止 `fail(String)` 和 `fail(int, String)` 等绕过登记源的方法。

## 2. 错误码治理

每个服务维护实现 `IErrorCode` 的 `XxxErrorCode` 枚举。新增或变更错误码须经评审，并校验全仓唯一性、六位长度、所属号段和 HTTP 状态。已发布错误码只能废弃，不得改语义或复用。

| 号段 | 所属组件 |
| --- | --- |
| `100000`–`109999` | 系统与网关 |
| `110000`–`119999` | 认证与授权组件 |
| `212000`–`212999` | iot-service |
| `213000`–`213999` | alert-service |
| `214000`–`214999` | care-service |
| `215000`–`215999` | vital-sign-service |
| `216000`–`216999` | family-living-service |
| `217000`–`217999` | operation-service |
| `218000`–`218999` | ai-service |

`200000`–`211999` 和 `219000` 以后是保留区，未经架构评审不得占用。

### 公共错误码

| 错误码 | HTTP | 稳定文案 | 用途 |
| --- | --- | --- | --- |
| `100001` | 500 | 系统繁忙，请稍后重试 | 未预期异常 |
| `100002` | 400 | 请求参数不合法 | 校验、类型转换、请求体格式 |
| `100003` | 405 | 请求方法不支持 | Method Not Allowed |
| `100004` | 415 | 不支持的内容类型 | Content-Type 不匹配 |
| `100005` | 429 | 请求过于频繁，请稍后再试 | 限流 |
| `100006` | 503 | 服务暂不可用，请稍后再试 | 本服务不可用 |
| `100007` | 502 | 依赖服务暂不可用，请稍后再试 | Feign/第三方依赖故障 |
| `100008` | 400 | 参数校验失败 | Bean Validation |
| `100009` | 404 | 请求资源不存在 | 公共组件通用资源不存在 |
| `100010` | 409 | 数据已存在或冲突 | 公共组件通用冲突 |
| `110001` | 401 | 登录状态已失效，请重新登录 | 未认证、Token 失效 |
| `110002` | 403 | 无权执行此操作 | 已认证但无权限 |

## 3. 异常分类

| 类型 | 产生位置 | 客户端响应 | 处理规则 |
| --- | --- | --- | --- |
| `BizException` | 领域规则、状态机、资源冲突 | 枚举绑定业务码和 HTTP 状态 | WARN，不自动重试 |
| 参数校验异常 | Controller 解析/校验 | `100002` 或 `100008` | 客户端修正后重试 |
| 认证/授权异常 | 网关或安全组件 | `110001` / `110002` | WARN，不泄露细节 |
| `RemoteCallException` | Feign、AI、第三方 SDK | `100007` 或明确只读降级 | 按幂等性重试 |
| 数据访问、MQ、序列化异常 | 基础设施层 | `100001` 或 `100006` | ERROR，由任务/MQ 策略处理 |
| 未知异常 | 兜底处理器 | `100001` | ERROR，保留完整堆栈 |

- `BizException` 只用于可预期业务失败，不得包装空指针、SQL 或第三方原始异常。
- Feign 超时、连接拒绝、熔断和非 2xx 响应统一转换为 `RemoteCallException`。
- 调用方只能返回 `100007`、明确的只读降级结果或中止流程；不得把 `null`、空集合或无时间戳缓存伪装成最新数据。
- `@RestControllerAdvice` 不处理 MQ 消费异常。消费者必须区分可重试故障与不可重试业务/数据错误。

## 4. 日志、审计与敏感信息

- 关键事件日志至少记录 `eventId`、`eventType`、`traceId`、业务主键、处理结果和错误码。
- 事件载荷、Token、密钥、证件号和手机号必须按规则脱敏。
- 签名或审计校验失败属于安全事件：保留证据、阻断敏感写操作并告警；不得向普通客户端暴露算法、密钥标识或校验细节。

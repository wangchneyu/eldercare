## 一、统一 REST 响应

所有对客户端公开的 REST 接口返回 `R<T>`；文件下载、流式响应、WebSocket 握手等协议例外必须在 OpenAPI 中声明。Feign 和 MQ 不使用 `R<T>` 作为内部传输契约。

```json
{
  "code": 0,
  "msg": "success",
  "data": {},
  "traceId": "01J..."
}
```

- `code`：业务码。`0` 是唯一成功码；失败必须是六位业务码，禁止使用 `500`、`404` 等 HTTP 数字代替。
- `msg`：面向用户的稳定、脱敏文案，由错误码定义；不返回堆栈、SQL、Token 或个人信息。
- `data`：成功时为业务数据；失败默认 `null`。参数校验失败可返回约定字段错误数组。
- `traceId`：网关生成或透传，并在日志、Feign Header、MQ Header 中保持一致。

### 1.1 HTTP 状态与业务码

HTTP 状态表示传输语义，`code` 表示业务语义。两者必须同时正确返回，不能用 HTTP 200 包装全部失败。

| HTTP 状态 | 使用场景 | 业务码范围/示例 |
| --- | --- | --- |
| 200 / 201 / 204 | 请求成功 | `0` |
| 400 | 参数、格式或业务前置条件不成立 | `100002`、具体业务码 |
| 401 | 未登录、Token 无效或过期 | `110001` |
| 403 | 已登录但无权限 | `110002` |
| 404 | 对外可见且允许暴露的资源不存在 | 对应服务资源不存在码 |
| 409 | 并发冲突、重复提交、非法状态流转 | 对应服务冲突码 |
| 429 | 限流 | `100005` |
| 502 / 503 | 下游或依赖不可用 | `100007` / `100006` |
| 500 | 未预期系统错误 | `100001` |

网关过滤器产生的 401、403、429、503 也必须返回同一 JSON 结构。WebFlux 网关使用响应式全局错误处理器；MVC 服务使用 `@RestControllerAdvice`，不得混用 Servlet API。

### 1.2 Java 骨架

```java
@Data
public class R<T> implements Serializable {
    private int code;
    private String msg;
    private T data;
    private String traceId;

    public static <T> R<T> ok(T data) {
        return result(data, 0, "success");
    }

    public static <T> R<T> fail(IErrorCode errorCode) {
        return result(null, errorCode.getCode(), errorCode.getMsg());
    }

    private static <T> R<T> result(T data, int code, String msg) {
        R<T> result = new R<>();
        result.setCode(code);
        result.setMsg(msg);
        result.setData(data);
        result.setTraceId(TraceContext.currentTraceId());
        return result;
    }
}
```

`TraceContext.currentTraceId()` 必须从 MDC/请求上下文读取；不得保留注释掉的赋值代码。

## 二、错误码模型与治理

```java
public interface IErrorCode {
    int getCode();
    String getMsg();
    HttpStatus getHttpStatus();
}
```

每个服务维护本服务的 `XxxErrorCode` 枚举并实现 `IErrorCode`。新增或变更错误码须经评审，CI 必须校验全仓唯一性、六位长度、所属号段和 HTTP 状态合法性。已发布错误码只能标记废弃，不得改语义或复用。

### 2.1 号段

| 号段 | 所属组件 | 说明 |
| --- | --- | --- |
| `100000`–`109999` | 系统与网关 | 参数/协议、限流、依赖不可用、未知错误 |
| `110000`–`119999` | 认证与授权组件 | 登录、Token、权限；不设独立 auth-service |
| `212000`–`212999` | iot-service | 设备、协议、指令与连接 |
| `213000`–`213999` | alert-service | SOS、预警、核销与 P0 状态机 |
| `214000`–`214999` | care-service | 档案、工单、交接班与护理状态 |
| `215000`–`215999` | vital-sign-service | 体征清洗、阈值和时序存储 |
| `216000`–`216999` | family-living-service | 日报、预约、账单和家属服务 |
| `217000`–`217999` | operation-service | 合同、排班、库存与运营 |
| `218000`–`218999` | ai-service | 推理、ASR/TTS 与模型能力 |

`200000`–`211999` 和 `219000` 以后为保留区，未经架构评审不得占用。

### 2.2 公共错误码

| 错误码 | HTTP | 文案 | 用途 |
| --- | --- | --- | --- |
| `100001` | 500 | 系统繁忙，请稍后重试 | 未预期异常 |
| `100002` | 400 | 请求参数不合法 | 校验、类型转换、请求体格式错误 |
| `100003` | 405 | 请求方法不支持 | Method Not Allowed |
| `100004` | 415 | 不支持的内容类型 | Content-Type 不匹配 |
| `100005` | 429 | 请求过于频繁，请稍后再试 | 网关/服务限流 |
| `100006` | 503 | 服务暂不可用，请稍后再试 | 本服务维护、熔断或不可用 |
| `100007` | 502 | 依赖服务暂不可用，请稍后再试 | Feign/第三方依赖故障 |
| `110001` | 401 | 登录状态已失效，请重新登录 | 未认证、Token 失效 |
| `110002` | 403 | 无权执行此操作 | 已认证但无权限 |

## 三、异常分类与使用规则

| 类型 | 产生位置 | 客户端响应 | 日志与重试 |
| --- | --- | --- | --- |
| `BizException` | 领域规则、状态机、资源冲突 | 枚举绑定的业务码与 HTTP 状态 | WARN；不自动重试业务拒绝 |
| 参数校验异常 | Controller 参数解析/校验 | `100002`，可带字段错误 | INFO/WARN；客户端修正后重试 |
| 认证/授权异常 | 网关或安全组件 | `110001` / `110002` | WARN；不泄露鉴权细节 |
| `RemoteCallException` | Feign、AI、第三方 SDK | `100007` 或明确只读降级结果 | WARN/ERROR；按幂等性重试 |
| 数据访问、MQ、序列化异常 | 基础设施层 | `100001` 或 `100006` | ERROR；由任务/MQ 策略处理 |
| 未知异常 | 兜底处理器 | `100001` | ERROR，保留完整堆栈 |

`BizException` 只用于可预期业务失败。禁止把空指针、SQL 或第三方原始异常包装成业务异常；禁止 catch 后记录日志再返回成功。

```java
public class BizException extends RuntimeException {
    private final IErrorCode errorCode;

    public BizException(IErrorCode errorCode) {
        super(errorCode.getMsg(), null, false, false);
        this.errorCode = errorCode;
    }

    public BizException(IErrorCode errorCode, Throwable cause) {
        super(errorCode.getMsg(), cause, true, true);
        this.errorCode = errorCode;
    }

    public IErrorCode getErrorCode() {
        return errorCode;
    }
}
```

默认构造器不填充堆栈，仅用于高频、预期业务拒绝；保留 cause 的构造器只用于确需追踪的转换边界。客户端文案不得用 `customMsg` 动态拼接，额外上下文仅进入脱敏日志。

### 3.1 MVC 全局处理器示例

```java
@RestControllerAdvice
@Slf4j
public class GlobalExceptionHandler {
    @ExceptionHandler(BizException.class)
    public ResponseEntity<R<Void>> handleBizException(BizException e) {
        IErrorCode code = e.getErrorCode();
        log.warn("biz error: code={}", code.getCode());
        return ResponseEntity.status(code.getHttpStatus()).body(R.fail(code));
    }

    @ExceptionHandler(Exception.class)
    public ResponseEntity<R<Void>> handleUnknown(Exception e) {
        log.error("unexpected error", e);
        return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR)
                .body(R.fail(SystemErrorCode.INTERNAL_ERROR));
    }
}
```

实际实现还必须处理请求体不可读、类型转换、参数校验、认证授权、限流和远程调用异常，并按本文表格映射。字段校验响应只包含 `field`、`reason`、脱敏后的 `rejectedValue`；不得回显密码、证件号等敏感值。

## 四、服务错误码示例

```java
public enum AlertErrorCode implements IErrorCode {
    SOS_EVENT_NOT_FOUND(213001, "SOS预警事件不存在", HttpStatus.NOT_FOUND),
    SOS_ALREADY_ASSIGNED(213002, "SOS工单已被其他护工抢单", HttpStatus.CONFLICT),
    NFC_VERIFY_FAILED(213003, "核销验证失败", HttpStatus.BAD_REQUEST),
    EVENT_SIGNATURE_MISMATCH(213005, "预警事件校验异常，请联系管理员", HttpStatus.SERVICE_UNAVAILABLE);
}

public enum CareErrorCode implements IErrorCode {
    ELDER_PROFILE_NOT_FOUND(214001, "长者健康档案不存在", HttpStatus.NOT_FOUND),
    TASK_ALREADY_COMPLETED(214002, "照护任务已完成，不可重复提交", HttpStatus.CONFLICT),
    HANDOVER_ALREADY_SUBMITTED(214004, "交接班记录已提交，不可重复编辑", HttpStatus.CONFLICT);
}
```

错误码表达业务结果，不表达底层原因。例如数据库超时不应新建“档案 SQL 超时”业务码，应记录根因并按 `100001` 或 `100006` 处理。

## 五、Feign、MQ 与审计边界

1. Feign 的超时、连接拒绝、熔断和非 2xx 响应统一转换为 `RemoteCallException`。调用方只能返回 `100007`、明确的只读降级结果，或中止流程；不得把 `null`、空集合或缓存伪装成最新数据。
2. MQ 消费者必须自行分类异常：可重试故障抛出以触发重试；不可重试业务/数据错误记录错误码后进入死信或人工处理。`@RestControllerAdvice` 不处理消息消费异常。
3. 关键事件日志至少记录 `eventId`、`eventType`、`traceId`、业务主键、处理结果和错误码。事件载荷、Token、密钥、证件号和手机号按脱敏规则记录。
4. 签名或审计校验失败属于安全事件：保留原始证据、阻断敏感写操作、告警管理员；不向普通客户端暴露签名算法、密钥标识或校验细节。

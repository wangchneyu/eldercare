# 公共组件使用手册

> 面向全体开发者，说明各公共组件**怎么用**、**怎么配置**、**有哪些坑**。

---

## 快速开始

### 1. 在 pom.xml 中引入依赖

所有公共组件版本由父 POM 统一管理，**子模块不需要写 `<version>`**。

```xml
<!-- ① 所有业务服务必引（统一响应体、异常处理、工具类） -->
<dependency>
    <groupId>com.eldercare</groupId>
    <artifactId>common-core</artifactId>
    <version>${project.version}</version>
</dependency>

<!-- ② 需要登录鉴权的服务 -->
<dependency>
    <groupId>com.eldercare</groupId>
    <artifactId>common-security</artifactId>
    <version>${project.version}</version>
</dependency>

<!-- ③ 需要 Redis 缓存 / 分布式锁的服务 -->
<dependency>
    <groupId>com.eldercare</groupId>
    <artifactId>common-redis</artifactId>
    <version>${project.version}</version>
</dependency>

<!-- ④ 需要调用其他微服务的服务 -->
<dependency>
    <groupId>com.eldercare</groupId>
    <artifactId>common-feign</artifactId>
    <version>${project.version}</version>
</dependency>

<!-- ⑤ 需要文件上传下载的服务 -->
<dependency>
    <groupId>com.eldercare</groupId>
    <artifactId>common-file</artifactId>
    <version>${project.version}</version>
</dependency>

<!-- ⑥ 需要操作审计日志的服务 -->
<dependency>
    <groupId>com.eldercare</groupId>
    <artifactId>common-audit</artifactId>
    <version>${project.version}</version>
</dependency>

<!-- ⑦ API 接口文档（所有 Web 服务都加） -->
<dependency>
    <groupId>com.eldercare</groupId>
    <artifactId>common-swagger</artifactId>
    <version>${project.version}</version>
</dependency>
```

### 2. application.yml 最小配置

```yaml
# 安全模块（引入 common-security 的服务必须配置）
eldercare:
  security:
    secret: ${ELDERCARE_JWT_SECRET}       # 环境变量注入，至少 32 字符
    whitelist:
      - /auth/login
      - /auth/register
      - /swagger-ui/**
      - /v3/api-docs/**

# 文件存储（引入 common-file 的服务按需配置，不配则默认本地存储）
  file:
    type: local                            # local | cos
```

## 一、common-core — 统一响应 & 异常 & 工具

### 1.1 接口统一返回 `R<T>`

```java
// 成功 — 无数据
return R.ok();

// 成功 — 有数据
return R.ok(userDTO);

// 失败 — 使用预定义错误码
return R.fail(SystemErrorCode.BAD_REQUEST);

// 失败 — 自定义消息
return R.fail(120001, "该老人已绑定照护计划");
```

**返回的 JSON 格式（所有接口统一）：**

```json
{
  "code": 0,
  "msg": "success",
  "data": { ... },
  "traceId": "a1b2c3d4e5f6..."
}
```

### 1.2 抛业务异常

```java
// 用预定义错误码
throw new BizException(SystemErrorCode.BAD_REQUEST);

// 用自定义错误码（各服务自行定义枚举实现 IErrorCode）
throw new BizException(CareErrorCode.CARE_PLAN_NOT_FOUND);
```

> **注意**：`BizException` 默认不填充堆栈，性能极高。如需堆栈用 `throw new BizException(errorCode, cause)`。

**全局异常处理器会自动捕获并转为 `R.fail(...)` 格式返回，你不需要手动 try-catch。**

### 1.3 自定义业务错误码

各服务在自己的包下创建枚举，实现 `IErrorCode` 接口：

```java
@Getter
@AllArgsConstructor
public enum CareErrorCode implements IErrorCode {
    CARE_PLAN_NOT_FOUND(120001, "照护计划不存在", HttpStatus.NOT_FOUND),
    CARE_PLAN_EXPIRED(120002, "照护计划已过期", HttpStatus.BAD_REQUEST);

    private final int code;
    private final String msg;
    private final HttpStatus httpStatus;
}
```

**错误码号段分配：**

| 号段 | 归属 |
|------|------|
| 100000–109999 | 系统/网关 |
| 110000–119999 | 认证/授权 |
| **120000–129999** | **service-iot** |
| **130000–139999** | **service-alert** |
| **140000–149999** | **service-care** |
| **150000–159999** | **service-vital** |
| **160000–169999** | **service-family** |
| **170000–179999** | **service-operation** |

### 1.4 常用工具类

```java
// ===== ID 生成 =====
Long id = IdUtil.nextId();              // Snowflake 全局唯一 ID
String uuid = IdUtil.uuid32();          // 32 位 UUID（无横线）
String bizId = IdUtil.prefixed("ORD");  // "ORD_xxxxxxxx-xxxx-..."

// ===== 数据脱敏 =====
DesensitizeUtil.mobilePhone("13812345678");  // "138****5678"
DesensitizeUtil.idCard("320101199001011234"); // "320101******1234"
DesensitizeUtil.name("张三");                 // "张*"

```

### 1.5 实体基类

所有数据库实体继承 `BaseEntity`：

```java
@Data
@EqualsAndHashCode(callSuper = true)
@TableName("t_care_plan")
public class CarePlan extends BaseEntity {
    // 只需写业务字段，id/createTime/updateTime/createBy/updateBy 已在基类
    private Long elderlyId;
    private String planName;
}
```

### 1.6 MQ 消息发送

```java
@Autowired
private MQProducer mqProducer;

// 发送消息
DeviceDataEvent event = new DeviceDataEvent();
event.fillEnvelope(MqTopicConstants.IOT_DEVICE_DATA, "service-iot:1.0.0");
event.setPayload(new DeviceDataPayload(deviceId, temperature));
mqProducer.send(MqTopicConstants.IOT_DEVICE_DATA, event);

// 延迟发送（delayLevel: 1=1s, 2=5s, 3=10s, 4=30s, 5=1m ...）
mqProducer.sendDelay(MqTopicConstants.ALERT_LOW_BATTERY, event, 4);
```

### 1.7 MQ 消息消费

```java
@Component
@RocketMQMessageListener(
    topic = MqTopicConstants.IOT_DEVICE_DATA,
    consumerGroup = "service-iot-device-data-consumer"
)
public class DeviceDataConsumer extends BaseConsumer<DeviceDataEvent>
        implements RocketMQListener<MessageExt> {

    public DeviceDataConsumer(ObjectMapper objectMapper) {
        super(objectMapper);
    }

    @Override
    public void onMessage(MessageExt messageExt) {
        super.onMessage(messageExt);  // 调用基类统一处理
    }

    @Override
    protected void processEvent(DeviceDataEvent event) {
        // 你的业务逻辑，这里已经自动：
        // - 反序列化了 payload
        // - 设置了 traceId 到 MDC
        // - 打印了消费日志
        DeviceDataPayload payload = event.getPayload();
        // ...
    }

    @Override
    protected Class<DeviceDataEvent> getEventClass() {
        return DeviceDataEvent.class;
    }
}
```

> **注意**：`consumerGroup` 命名规范为 `{服务名}-{topic}-consumer`，每个消费者组必须唯一。

## 二、common-security — 登录鉴权

### 2.1 获取当前登录用户

```java
// 方式一：获取完整用户信息
LoginUser user = SecurityContextHolder.getLoginUser();
Long userId = user.getUserId();
String username = user.getUsername();
Set<UserRole> roles = user.getRoles();

// 方式二：只获取用户 ID（最常用）
Long userId = SecurityContextHolder.getCurrentUserId();
```

### 2.2 限制接口角色

```java
// 只有管理员能访问
@RequireRole(UserRole.ADMIN)
@DeleteMapping("/user/{id}")
public R<Void> deleteUser(@PathVariable Long id) { ... }

// 管理员或护理员都能访问
@RequireRole({UserRole.ADMIN, UserRole.CAREGIVER})
@PutMapping("/care-plan/{id}")
public R<Void> updateCarePlan(@PathVariable Long id, ...) { ... }
```

### 2.3 签发 JWT（登录接口用）

```java
@Autowired
private JwtTokenProvider jwtTokenProvider;

@PostMapping("/auth/login")
public R<JwtTokenPair> login(@RequestBody LoginDTO dto) {
    LoginUser loginUser = authenticate(dto);  // 你的认证逻辑
    JwtTokenPair tokenPair = jwtTokenProvider.createTokenPair(loginUser);
    return R.ok(tokenPair);
}
```

### 2.4 配置白名单路径

不需要登录就能访问的接口，加到白名单：

```yaml
eldercare:
  security:
    whitelist:
      - /auth/login
      - /auth/register
      - /auth/refresh-token
      - /swagger-ui/**
      - /v3/api-docs/**
      - /actuator/health
```

### 2.5 关闭安全模块（仅开发调试用）

```yaml
eldercare:
  security:
    enabled: false
```

## 三、common-redis — 缓存 & 分布式锁

### 3.0 Redis 连接配置

引入 `common-redis` 后，必须在 `application.yml` 中配置 Redis 地址：

```yaml
spring:
  data:
    redis:
      host: ${REDIS_HOST:localhost}
      port: ${REDIS_PORT:6379}
      password: ${REDIS_PASSWORD:}        # 无密码则留空
      database: 0
      lettuce:
        pool:
          max-active: 16
          max-idle: 8
          min-idle: 2
          max-wait: 3000ms
```

> 开发环境默认连 `localhost:6379`，生产环境通过环境变量注入。

### 3.1 缓存操作

```java
@Autowired
private RedisService redisService;

// 存取
redisService.set("user:1001", userDTO);
redisService.set("user:1001", userDTO, 3600);  // 过期时间 3600 秒
UserDTO user = redisService.get("user:1001");

// 删除
redisService.delete("user:1001");

// 计数器
redisService.increment("sms:code:count:13800001111");
Long count = redisService.increment("order:daily:count", 1);

// Hash
redisService.hSet("device:1001", "temperature", 36.5);
Object temp = redisService.hGet("device:1001", "temperature");

// 判断 key 是否存在
Boolean exists = redisService.hasKey("user:1001");
```

### 3.2 分布式锁

```java
@Autowired
private RedisLock redisLock;

public void createOrder(OrderDTO dto) {
    String lockKey = "lock:order:" + dto.getElderlyId();
    String requestId = IdUtil.uuid32();  // 每次请求唯一标识

    try {
        // 尝试获取锁，持有 30 秒，最多等待 5 秒
        boolean locked = redisLock.tryLock(lockKey, requestId, 30, 5);
        if (!locked) {
            throw new BizException(SystemErrorCode.TOO_MANY_REQUESTS);
        }

        // 业务逻辑 ...

    } finally {
        // 必须在 finally 中释放锁
        redisLock.unlock(lockKey, requestId);
    }
}
```

> **注意**：`requestId` 必须每次请求唯一，推荐用 `IdUtil.uuid32()`。锁的 `leaseTime` 要大于业务执行时间。

### 3.3 Key 命名规范

```
{业务域}:{实体}:{ID}

示例：
  user:info:1001          # 用户信息
  device:telemetry:2001   # 设备遥测
  alert:pending:3001      # 待处理告警
  lock:order:4001         # 订单锁
  sms:code:13800001111    # 短信验证码
```

## 四、common-feign — 微服务间调用

### 4.1 调用其他服务

```java
@Autowired
private AlertClient alertClient;

public void processAlert(Long alertId) {
    try {
        AlertRemoteDTO alert = alertClient.getAlert(alertId);
        // 处理告警 ...
    } catch (RemoteCallException e) {
        // alert 服务不可用，做降级处理
        log.warn("告警服务暂不可用，稍后重试");
    }
}
```

### 4.2 启动类加注解

```java
@SpringBootApplication
@EnableFeignClients(basePackages = "com.eldercare.common.feign.client")
public class ServiceCareApplication {
    public static void main(String[] args) {
        SpringApplication.run(ServiceCareApplication.class, args);
    }
}
```

### 4.3 注意事项

- Feign 调用会**自动透传**当前用户的身份信息（X-User-Id / X-User-Roles）和链路追踪 ID（X-Trace-Id），你不需要手动处理
- 默认超时：连接 5s / 读取 10s
- **默认不重试**（由 Sentinel 熔断控制），不要在 Feign 层加重试
- 调用失败会抛 `RemoteCallException`，全局异常处理器会返回 502

## 五、common-file — 文件上传下载

### 5.1 上传文件

```java
@Autowired
private IFileStorageService fileStorageService;

@PostMapping("/upload")
public R<String> upload(@RequestParam("file") MultipartFile file) {
    String fileId = fileStorageService.upload(
        file.getInputStream(),
        file.getOriginalFilename(),
        file.getContentType()
    );
    return R.ok(fileId);
}
```

### 5.2 获取预览 URL

```java
@GetMapping("/preview")
public R<String> preview(@RequestParam String fileId) {
    String url = fileStorageService.getPreviewUrl(fileId, 30);  // 30 分钟有效
    return R.ok(url);
}
```

### 5.3 切换存储方式

```yaml
# 本地存储（开发环境默认）
eldercare:
  file:
    type: local
    local:
      upload-path: ./uploads/
      domain: http://localhost:8080

# 腾讯云 COS（生产环境）
eldercare:
  file:
    type: cos
    cos:
      secret-id: ${COS_SECRET_ID}
      secret-key: ${COS_SECRET_KEY}
      region: ap-guangzhou
      bucket-name: eldercare-xxx
      domain: https://cdn.eldercare.cn
```

## 十、常见问题 FAQ

### Q: 接口返回格式不统一怎么办？

A: 所有 Controller 方法返回类型统一用 `R<T>`，不要用 `ResponseEntity` 或裸对象。全局异常处理器会自动处理异常。

### Q: 怎么获取当前登录用户 ID？

A: `SecurityContextHolder.getCurrentUserId()`。如果返回 null，说明该接口没有经过鉴权（在白名单中或安全模块未启用）。

### Q: Feign 调用报 `RemoteCallException` 怎么排查？

A: 查看日志中的 `service` 和 `path` 字段，确认目标服务是否在 Nacos 注册、网络是否可达。

### Q: Redis 分布式锁获取失败怎么处理？

A: 返回 false 时根据业务场景决定：抛 `BizException(SystemErrorCode.TOO_MANY_REQUESTS)` 提示用户稍后重试，或做降级处理。

### Q: 自定义错误码怎么定义？

A: 各服务创建自己的 `XxxErrorCode` 枚举，实现 `IErrorCode` 接口。号段按上方分配表取，避免冲突。

### Q: 开发环境不想配 JWT 密钥怎么办？

A: 在 `application.yml` 中设置 `eldercare.security.enabled: false` 关闭安全模块。**生产环境必须开启**。

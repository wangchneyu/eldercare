# 公共组件（Common）设计说明

## 模块总览

```
common/
├── common-core        # 核心基础模块 — 全局响应体、异常处理、工具类、MQ 基础设施
├── common-security    # 安全认证模块 — JWT 签发/验签、鉴权过滤器、角色权限 AOP
├── common-redis       # Redis 基础模块 — 缓存、分布式锁、Snowflake 工作机器 ID 分配
├── common-feign       # Feign 远程调用模块 — 服务间 Client、Fallback、超时配置
├── common-file        # 文件存储模块 — 本地/腾讯云 COS 对象存储抽象
├── common-notify      # 通知推送模块 — 短信、语音、微信订阅消息
├── common-audit       # 审计日志模块 — 操作日志注解 + AOP 切面
├── common-swagger     # 接口文档模块 — SpringDoc OpenAPI 自动配置
└── common-notification# （预留）邮件通知模块，暂未实现
```

### 模块依赖关系

```
common-core
├── common-security    → 依赖 common-core
├── common-redis       → 依赖 common-core
├── common-feign       → 依赖 common-core
├── common-file        → 依赖 common-core
├── common-notify      → 依赖 common-core
├── common-audit       → 依赖 common-core
└── common-swagger     → 无依赖（独立）
```

## 1. common-core — 核心基础模块

> 全局响应体、异常体系、工具类、枚举常量、MQ 生产/消费基础设施

### 1.1 通用响应体 `R<T>`

| 类 | 说明 |
|---|---|
| `R<T>` | 统一 API 响应封装，包含 `code`、`msg`、`data`、`traceId` 四个字段 |

- 所有接口返回统一 JSON 格式：`{"code":0,"msg":"success","data":{},"traceId":"..."}`
- `traceId` 由 `TraceContext` 自动填充，确保全链路可追踪
- 工厂方法：`R.ok()` / `R.ok(data)` / `R.fail(errorCode)` / `R.fail(code, msg)`

### 1.2 异常体系

| 类/接口 | 说明 |
|---|---|
| `IErrorCode` | 错误码接口，定义 `getCode()`、`getMsg()`、`getHttpStatus()` |
| `SystemErrorCode` | 系统级错误码枚举（100001–119999 号段） |
| `BizException` | 业务异常，不填充堆栈（高性能），携带 `IErrorCode` |
| `RemoteCallException` | 远程调用异常（Feign 降级时抛出） |
| `GlobalExceptionHandler` | MVC 全局异常处理器，覆盖参数校验/类型转换/业务异常/远程调用/未知异常 |
| `SecurityExceptionHandler` | 认证授权异常处理器（403 越权等） |

**错误码号段规划：**

| 号段 | 分类 |
|---|---|
| 100000–109999 | 系统与网关 |
| 110000–119999 | 认证与授权 |
| 120000+ | 各业务服务自定义 |

### 1.3 领域基类

| 类 | 说明 |
|---|---|
| `BaseEntity` | 实体基类，包含 `id`（Snowflake Long）、`createTime`、`updateTime`、`createBy`、`updateBy` |
| `BaseEvent<T>` | MQ 事件基类，封装事件信封（eventId、eventType、schemaVersion、occurredAt、traceId、producer、payload） |

### 1.4 枚举常量

| 枚举 | 说明 |
|---|---|
| `RoleType` | 人员账号角色（SUPER_ADMIN / ORG_ADMIN / CARE_MANAGER / CAREGIVER / DOCTOR / FAMILY_MEMBER / ELDERLY） |
| `AlarmLevel` | 告警级别 |
| `CareLevel` | 照护等级 |
| `DeviceType` | 设备类型 |
| `EventSource` | 事件来源 |
| `TaskStatus` | 任务状态 |

### 1.5 工具类

| 类 | 说明 |
|---|---|
| `IdUtil` | 全局唯一 ID 生成：Snowflake（10位 workerId + 12位 sequence）、UUID、带前缀 ID |
| `DateTimeUtil` | 日期时间工具，默认时区 `Asia/Shanghai`，格式化/解析/过期判断 |
| `DesensitizeUtil` | 数据脱敏：手机号、身份证、姓名、邮箱、银行卡 |
| `TraceContext` | 链路追踪上下文，基于 SLF4J MDC 实现 traceId 的存取与生成 |
| `TraceableAsyncConfigurer` | 异步线程池配置，自动传递 traceId 到子线程 |
| `TraceableExecutorDecorator` | Executor 装饰器，跨线程传递 MDC 上下文 |

### 1.6 MQ 基础设施（RocketMQ）

| 类 | 说明 |
|---|---|
| `MQProducer` | 消息生产者封装，支持同步/异步/延迟三种发送模式 |
| `BaseConsumer<T>` | 消息消费者抽象基类，封装反序列化、traceId 提取、幂等检查、消费日志 |
| `RocketMQAutoConfiguration` | RocketMQ 自动配置 |
| `RocketMQProperties` | RocketMQ 可配置属性 |
| `MqTopicConstants` | MQ Topic 常量定义（命名规范：`{domain}_{event_type}_{version}`） |

**消息规范：**
- 消息体：`BaseEvent` JSON 序列化
- 消息 Key：`eventId`（支持幂等和顺序消息）
- User Properties：`X-Trace-Id`（链路追踪）
- 消费失败自动重试 16 次，超过后进入死信队列

## 2. common-security — 安全认证模块

> JWT 签发/验签/刷新、鉴权过滤器、角色权限 AOP、Feign 拦截器

### 2.1 核心组件

| 类 | 说明 |
|---|---|
| `JwtTokenProvider` | JWT 令牌提供者：签发 Access/Refresh Token、验签、解析 Claims、刷新令牌 |
| `JwtAuthenticationFilter` | 鉴权过滤器（`OncePerRequestFilter`），支持两种模式 |
| `SecurityContextHolder` | 安全上下文持有者，基于 ThreadLocal 存储当前请求用户信息 |
| `SecurityAutoConfiguration` | 安全模块自动配置（SPI 注册，可通过 `eldercare.security.enabled=false` 关闭） |
| `SecurityProperties` | 安全配置属性 |

### 2.2 鉴权过滤器双模式

1. **网关透传模式（高性能）**：优先读取 `X-User-Id` / `X-User-Name` / `X-User-Roles` 请求头
2. **本地验签模式（回退）**：无透传头时从 `Authorization` 提取 JWT 本地解密

### 2.3 角色权限

| 类 | 说明 |
|---|---|
| `UserRole` | 用户角色枚举（ADMIN / FAMILY / CAREGIVER / OPERATOR / DOCTOR） |
| `LoginUser` | 登录用户信息（userId、username、roles） |
| `@RequireRole` | 角色权限注解，声明方法所需角色 |
| `RequireRoleAspect` | AOP 切面，校验当前用户是否拥有注解声明的角色 |

### 2.4 Feign 拦截器

| 类 | 说明 |
|---|---|
| `FeignAuthRequestInterceptor` | Feign 调用拦截器，自动透传用户身份（X-User-*）和链路追踪（X-Trace-Id）到下游服务 |

### 2.5 配置项

```yaml
eldercare:
  security:
    enabled: true                              # 是否启用安全模块
    secret: ${ELDERCARE_JWT_SECRET}            # JWT 签名密钥（最小 32 字符）
    access-token-expiration: 7200              # 访问令牌过期时间（秒），默认 2 小时
    refresh-token-expiration: 604800           # 刷新令牌过期时间（秒），默认 7 天
    whitelist:                                 # 鉴权白名单（Ant 风格路径）
      - /auth/login
      - /auth/register
```

## 3. common-redis — Redis 基础模块

> Redis 配置、缓存操作、分布式锁、Snowflake 工作机器 ID 动态分配

### 3.1 核心组件

| 类 | 说明 |
|---|---|
| `RedisConfig` | RedisTemplate 自动配置，Key 使用 String 序列化，Value 使用 JSON（GenericJackson2JsonRedisSerializer） |
| `RedisAutoConfiguration` | 模块自动配置入口 |
| `RedisService` | Redis 通用操作服务（String/Hash/Set/List 全覆盖） |
| `RedisCache` | Redis 缓存操作工具类（与 RedisService 功能互补） |
| `RedisLock` | 基于 Redis 的轻量级分布式锁（SETNX + Lua 脚本原子解锁） |
| `RedisWorkerIdAllocator` | 基于 Redis 的 Snowflake workerId 动态分配器（心跳续期 + 自动释放） |

### 3.2 分布式锁使用方式

```java
// 非阻塞模式
boolean locked = redisLock.tryLock("order:lock:123", requestId, 30);

// 阻塞模式（等待 5 秒）
boolean locked = redisLock.tryLock("order:lock:123", requestId, 30, 5);

// 释放锁（Lua 脚本原子操作）
redisLock.unlock("order:lock:123", requestId);
```

### 3.3 Snowflake WorkerId 分配机制

1. 启动时遍历 1024 个槽位，通过 `SETNX` 竞争分配
2. 每 20 秒心跳续期（TTL 60 秒），防止实例宕机后槽位被回收
3. 实例关闭时 `@PreDestroy` 主动释放槽位
4. 分配成功后自动初始化 `IdUtil` 的 Snowflake 实例

## 4. common-feign — Feign 远程调用模块

> 服务间 Client 定义、Fallback 降级工厂、超时与重试配置

### 4.1 服务客户端

| Client | 目标服务 | 说明 |
|---|---|---|
| `AlertClient` | service-alert | 告警服务 |
| `AuthClient` | service-operation | 运营/鉴权服务 |
| `CareClient` | service-care | 照护服务 |
| `FamilyClient` | service-family | 家属服务 |
| `IotClient` | service-iot | 物联网服务 |
| `OperationClient` | service-operation | 运营服务 |
| `VitalClient` | service-vital | 体征监测服务 |

### 4.2 Fallback 降级

每个 Client 对应一个 `FallbackFactory`，降级时抛出 `RemoteCallException`，由 `GlobalExceptionHandler` 统一处理返回 502 响应。

### 4.3 超时与重试策略

| 配置项 | 默认值 | 说明 |
|---|---|---|
| `connectTimeout` | 5000ms | 连接超时 |
| `readTimeout` | 10000ms | 读取超时 |
| `retryer` | `NEVER_RETRY` | 不重试（由 Sentinel 熔断控制，避免雪崩） |

### 4.4 DTO 定义

| DTO | 说明 |
|---|---|
| `AlertRemoteDTO` | 告警信息 |
| `UserRemoteDTO` | 用户信息 |
| `CarePlanRemoteDTO` | 照护计划 |
| `FamilyMemberRemoteDTO` | 家庭成员 |
| `DeviceTelemetryRemoteDTO` | 设备遥测数据 |
| `OperationRemoteDTO` | 运营数据 |
| `VitalRecordRemoteDTO` | 体征记录 |

## 5. common-file — 文件存储模块

> 文件上传/下载/预览/删除的统一抽象，支持本地存储和腾讯云 COS

### 5.1 核心接口

```java
public interface IFileStorageService {
    String upload(InputStream inputStream, String originalFilename, String contentType);
    InputStream download(String fileIdOrPath);
    String getPreviewUrl(String fileIdOrPath, int expiryMinutes);
    void delete(String fileIdOrPath);
}
```

### 5.2 实现类

| 实现 | 触发条件 | 说明 |
|---|---|---|
| `LocalFileStorageServiceImpl` | `eldercare.file.type=local`（默认） | 本地文件系统存储 |
| `CosFileStorageServiceImpl` | `eldercare.file.type=cos` | 腾讯云 COS 对象存储 |

### 5.3 配置项

```yaml
eldercare:
  file:
    type: local                    # 存储类型: local | cos
    local:
      upload-path: ./uploads/      # 本地存储路径
      domain: http://localhost:8080
    cos:
      secret-id: ${COS_SECRET_ID}
      secret-key: ${COS_SECRET_KEY}
      region: ap-guangzhou
      bucket-name: eldercare-xxx
      domain: https://cdn.eldercare.cn
```

## 6. common-notify — 通知推送模块

> 短信(SMS)、语音(Voice)、微信(WeChat)推送通道抽象

### 6.1 核心接口

```java
public interface INotifyService {
    void sendSms(SmsRequest request);
    void sendWeChat(WeChatRequest request);
    void sendVoice(VoiceRequest request);
}
```

### 6.2 实现类

| 实现 | 说明 |
|---|---|
| `AliyunSmsNotifyServiceImpl` | 阿里云短信/语音通道实现 |
| `MockNotifyServiceImpl` | Mock 实现（开发调试用，仅打印日志） |

### 6.3 DTO

| DTO | 说明 |
|---|---|
| `SmsRequest` | 短信请求（手机号、模板ID、参数） |
| `VoiceRequest` | 语音呼叫请求 |
| `WeChatRequest` | 微信订阅消息请求 |

> **注意**：邮件(Email)通知不在本模块范围内。

## 7. common-audit — 审计日志模块

> 基于注解 + AOP 的操作审计日志

### 7.1 使用方式

```java
@AuditLog(value = "修改照护计划", saveParams = true, saveResult = false)
@PutMapping("/care-plan/{id}")
public R<Void> updateCarePlan(@PathVariable Long id, @RequestBody CarePlanDTO dto) {
    // ...
}
```

### 7.2 审计日志内容

切面自动采集以下信息并输出结构化日志：

| 字段 | 说明 |
|---|---|
| `operation` | 操作描述（注解 value） |
| `traceId` | 链路追踪 ID |
| `userId` / `username` | 操作人（从网关透传请求头提取） |
| `clientIp` | 客户端 IP（支持代理穿透） |
| `url` / `httpMethod` | 请求路径和方法 |
| `class` / `method` | 目标类和方法 |
| `duration` | 执行耗时 |
| `params` | 请求参数（可配置是否保存） |
| `result` | 响应结果（可配置是否保存） |
| `status` | 执行状态（SUCCESS / FAIL） |

## 8. common-swagger — 接口文档模块

> SpringDoc OpenAPI 自动配置，各服务零代码接入

### 8.1 接入方式

引入依赖即可，自动读取 `spring.application.name` 和 `server.servlet.context-path` 生成文档。

### 8.2 功能特性

- 自动生成 OpenAPI 3.0 规范文档
- 内置 Bearer JWT 认证方案
- 支持网关入口 + 本地直连两种 Server 地址
- Swagger UI 访问路径：`{context-path}/swagger-ui.html`
